# Verification Layer for MedScholar Agent

This verification layer provides metrics to assess the factual reliability of generated literature survey reports.

## Overview

The verification system is designed to run **after** a literature survey report has been generated by the MedScholar agent. It analyzes the report and calculates various metrics to assess its quality and reliability.

## Current Metrics

### 1. Citation Accuracy ✓ (Implemented)

Measures the validity and accessibility of citations in the report.

**Checks:**

- URL Format: Whether URLs are properly formatted
- URL Accessibility: Whether cited URLs are reachable (HTTP 200-399)
- Source Credibility: Whether URLs are from trusted academic/medical sources
- Suspicious Patterns: Detects fake URLs (example.com, localhost, etc.)

**Score Calculation:**
Multi-factor weighted scoring:

- Accessibility: 40%
- Credibility: 30%
- Uniqueness: 20%
- Non-suspicious: 10%

See [citation_accuracy/README.md](citation_accuracy/README.md) for detailed documentation.

### 2. Claim Support Rate ✓ (Implemented)

Measures the percentage of factual claims that have inline citation markers.

**Checks:**

- Extracts all factual claims from the report
- Identifies claims with inline citations [1], [2], [3]
- Calculates support percentage

**Score Calculation:**

```
Claim Support Rate = (Claims with Citations / Total Claims)
```

A claim is considered supported if it contains at least one inline citation marker like [1], [2], etc.

See [claim_support/README.md](claim_support/README.md) for detailed documentation.

### 3. Evidence Match Quality ✓ (Implemented)

Measures whether cited sources actually support the claims made in the report.

**Checks:**

- Fetches and extracts text from cited URLs
- Calculates TF-IDF similarity between claims and source content
- Performs keyword matching analysis
- Identifies best matching passages in sources

**Score Calculation:**
Multi-factor weighted scoring:

- TF-IDF Similarity: 50%
- Keyword Match Rate: 30%
- Best Passage Match: 20%

**Match Quality Levels:**

- **HIGH** (≥0.7): Strong evidence match - source clearly supports claim
- **MEDIUM** (0.4-0.69): Partial match - source discusses topic but support is weak
- **LOW** (<0.4): Poor match - source doesn't adequately support claim

See [evidence_match/README.md](evidence_match/README.md) for detailed documentation.

### 4. Hallucination Rate (Planned)

Will identify claims that are not grounded in any retrieved document.

## Usage

### Verify a Single Report

```bash
python run_verification.py --report workspace/your_report.md
```

This will:

1. Parse the report
2. Run all verification metrics
3. Save results to `workspace/your_report_verification.json`

### Verify All Reports in Workspace

```bash
python run_verification.py --all
```

This will:

1. Find all `.md` and `.txt` files in the `workspace/` directory
2. Verify each report
3. Save individual results as `{report_name}_verification.json`
4. Create a summary file `workspace/verification_summary.json`

### Custom Options

```bash
# Specify custom workspace directory
python run_verification.py --all --workspace my_reports/

# Custom output path for single report
python run_verification.py --report workspace/report.md --output results/verification.json

# Skip URL accessibility checks (faster, less thorough)
python run_verification.py --report workspace/report.md --no-check-urls

# Custom timeout for URL checks (default: 10 seconds)
python run_verification.py --report workspace/report.md --timeout 15
```

## Adding New Metrics

To add a new verification metric:

1. Create a new file in `app/verification/` (e.g., `claim_support.py`)

2. Extend the `VerificationMetric` base class:

```python
from app.verification.base import VerificationMetric, VerificationResult

class ClaimSupportMetric(VerificationMetric):
    def __init__(self):
        super().__init__("Claim Support Rate")

    async def calculate(self, report_path: str, **kwargs) -> VerificationResult:
        # Your implementation here
        pass
```

3. Add to `app/verification/__init__.py`:

```python
from app.verification.claim_support import ClaimSupportMetric

__all__ = [..., "ClaimSupportMetric"]
```

4. Add to the metrics list in `run_verification.py`:

```python
self.metrics = [
    CitationAccuracyMetric(),
    ClaimSupportMetric(),  # Add here
]
```

## Performance Considerations

- **Citation Accuracy**: URL checking is parallelized using `asyncio.gather()` for speed
- **Timeout**: Default 10 seconds per URL (configurable with `--timeout`)
- **Caching**: Future versions may cache URL accessibility results
- **Rate Limiting**: Consider adding delays for large numbers of URLs to avoid overwhelming servers

## Troubleshooting

### Issue: SSL Certificate Errors

The citation checker disables SSL verification by default (`ssl=False`) to avoid certificate issues with some academic sites. This is acceptable for verification purposes but should not be used for security-critical applications.

### Issue: Timeouts

Some URLs may be slow to respond. Increase the timeout:

```bash
python run_verification.py --report workspace/report.md --timeout 30
```

### Issue: No Citations Found

If no citations are found, check that your report has a "References" section with properly formatted citations:

```markdown
## References

1. [Title](https://url.com)
2. Title. Available at: [Source](https://url.com)
```
